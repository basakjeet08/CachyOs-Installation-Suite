# Docker Setup Guidelines

- This is a guide to how to install docker in your system. Please refer to [Phabricator](https://phab.instahyre.com/w/docker/) for more details on the process.

## Install Docker and Docker Compose

- Check if you already have them installed

```bash
pacman -Q docker docker-compose
```

- If they are not yet installed then install them

```bash
sudo pacman -S docker docker-compose
```

## Enable and start Docker Services

- Need to make sure docker is always started after a reboot

```bash
sudo systemctl enable --now docker.service
```

## Add your user to the Docker group

- Need to add user to make sure we can run docker commands without sudo

```bash
sudo usermod -aG docker $USER
```

- Log out and log back in, or restart your shell:

```bash
exec fish
```

- Verify if you are able to run docker without sudo permissions. This may require a system restart

```bash
docker ps
```

## Now build the docker images and containers

- We are not shifting everything to docker since that makes debugging the project harder. What we will do is have our `RabbitMQ`, `MySQL`, `ElasticSearch` and `Redis` Setups inside docker while we install `python` and the `Django Codebase` in our local system.

- To do that you need to open this file inside `/devel/docker/docker-compose.yml` file and replace the contents with these.

```yml
version: "3.8"

name: instahyre-devel

services:
  mysql:
    image: mysql:8.0
    hostname: rds.instahyre.internal
    container_name: mysql
    restart: always
    environment:
      MYSQL_ALLOW_EMPTY_PASSWORD: "yes"
    volumes:
      - mysql-data:/var/lib/mysql
    ports:
      - "3306:3306"
    networks:
      - instahyre-local
    healthcheck:
      test: ["CMD-SHELL", "mysqladmin ping -h localhost -u root || exit 1"]
      interval: 5s
      timeout: 10s
      retries: 3
      start_period: 20s

  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.10.2
    container_name: elasticsearch
    restart: always
    hostname: es.instahyre.internal
    environment:
      - node.name=es-node
      - cluster.name=es-cluster
      - discovery.type=single-node
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - es-data:/usr/share/elasticsearch/data
    ports:
      - "9200:9200"
    networks:
      - instahyre-local
    healthcheck:
      test: curl -s http://localhost:9200/_cluster/health || exit 1
      interval: 10s
      timeout: 5s
      retries: 3

  rabbitmq:
    image: rabbitmq:3.9.13-management-alpine
    hostname: mq.instahyre.internal
    container_name: rabbitmq
    restart: always
    environment:
      RABBITMQ_DEFAULT_USER: "blobby"
      RABBITMQ_DEFAULT_PASS: "blobby"
      RABBITMQ_DEFAULT_VHOST: "/"
    ports:
      - "5672:5672"
      - "15672:15672"
    volumes:
      - rabbitmq-data:/var/lib/rabbitmq
    networks:
      - instahyre-local
    healthcheck:
      test: rabbitmq-diagnostics -q ping
      interval: 5s
      timeout: 10s
      retries: 3

  redis:
    image: redis:6.0.3
    hostname: cache.instahyre.internal
    container_name: redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - instahyre-local
    healthcheck:
      test: redis-cli ping || exit 1
      interval: 5s
      timeout: 10s
      retries: 3

volumes:
  mysql-data:
  es-data:
  rabbitmq-data:
  redis-data:

networks:
  instahyre-local:
    driver: bridge
```

- After that use the new docker compose file and build the images and start them

```bash
docker compose -f /home/path_to_docker/docker-compose.yml up -d
```

- Check if the containers are up. If yes then its done

```bash
docker ps
```

## If the contianers are not started try these steps below

- After that, build the docker images

```bash
docker-compose up --no-start --build

# If the above command doesn't work then try this
docker compose up --not-start --build
```

- Start the docker containers

```bash
docker compose start
```

- After this is done please revert back the original contents inside `docker-compose.yml file`

## Taking the latest DB Dump and adding it to SQL Container

- Check if you are able to connect to the DB container.

```bash
docker exec -it mysql mysql -u root
```

- Check if the `auction` database exists.

```bash
docker exec -it mysql mysql -u root -e "SHOW DATABASES;"
```

- If no database exists then create the `auction` DB.

```bash
docker exec -it mysql mysql -u root -e "CREATE DATABASE auction;"
```

- Start importing the DB Dump.

```bash
pv ~/backup.sql | docker exec -i mysql mysql -u root auction
```

## If facing issues where sql is rejecting the connection then run

- Open the SQL Container

```bash
docker exec -it mysql mysql -u root
```

- Run this command inside your SQL Shell

```bash
CREATE USER 'root'@'%' IDENTIFIED BY 'root';
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;
```
